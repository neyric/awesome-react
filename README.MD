### UI Libraries
*Pre-built UI component libraries and design systems for building React applications.*

- [Shadcn UI](https://github.com/shadcn-ui/ui) - A collection of accessible, copy-pasteable UI components built with Radix UI and Tailwind CSS.
- [Mantine UI](https://github.com/mantinedev/mantine) - A fully featured component library with built-in dark mode and advanced customization hooks.
- [Daisy UI](https://github.com/saadeghi/daisyui) - A popular plugin for Tailwind CSS that adds semantic component classes.
- [Ant Design](https://github.com/ant-design/ant-design) - An enterprise-grade UI design language and component library.
- [Material UI](https://github.com/mui/material-ui) - A comprehensive component library implementing Google's Material Design system.
- [Chakra UI](https://github.com/chakra-ui/chakra-ui) - A simple, modular, and accessible component library focused on developer experience.
- [React Bootstrap](https://github.com/react-bootstrap/react-bootstrap) - The most popular front-end framework rebuilt for React without jQuery dependencies.
- [Fluent UI](https://github.com/microsoft/fluentui) - A collection of UX frameworks for building web apps that fit seamlessly into the Microsoft ecosystem.
- [Framework7](https://github.com/framework7io/framework7) - A full-featured framework for building mobile and hybrid apps with a native look and feel.
- [Ariakit](https://github.com/ariakit/ariakit) - A low-level library for building accessible, WAI-ARIA compliant web components.
- [8bitcn UI](https://github.com/TheOrcDev/8bitcn-ui) - A retro-styled, 8-bit component library for nostalgic user interfaces.

### Frameworks
*Full-stack and frontend frameworks built on top of React for building modern web applications.*

- [Next.JS](https://github.com/vercel/next.js) - The leading framework for building server-rendered and static React applications.
- [Remix](https://github.com/remix-run/remix) - A full-stack framework focused on web standards and modern user experience.
- [Gatsby](https://github.com/gatsbyjs/gatsby) - A fast, open-source static site generator built on React and GraphQL.
- [Tanstack Start](https://github.com/tanstack/router) - A full-stack React framework powered by TanStack Router for type-safe routing and server actions.
- [Vike](https://github.com/vikejs/vike) - A modular, flexible framework for building SSR and static apps with any UI library.

### Hooks & Data Fetching
*Libraries for managing application state and fetching data in React applications.*

- [Tanstack Query](https://github.com/TanStack/query) - Powerful asynchronous state management and data fetching for the web.
- [SWR](https://github.com/vercel/swr) - A lightweight React Hook for data fetching focused on stale-while-revalidate caching.
- [useHooks](https://github.com/uidotdev/usehooks) - A collection of modern, server-safe React Hooks for common utilities.
- [aHooks](https://github.com/alibaba/hooks) - A comprehensive library of high-quality and reliable React Hooks.
- [Mantine Hooks](https://mantine.dev/hooks/package/) - A standalone package of utility hooks for state and UI management.

### State Management
*Libraries for managing global application state.*

- [Zustand](https://github.com/pmndrs/zustand) - A small, fast, and scalable barebones state management solution.
- [Redux](https://github.com/reduxjs/redux) - A predictable state container for JavaScript apps, often used for global state management.
- [Jotai](https://github.com/pmndrs/jotai) - A primitive and flexible atomic state management library.
- [Mobx](https://github.com/mobxjs/mobx) - Simple, scalable state management using functional reactive programming.
- [Recoil](https://github.com/facebookexperimental/Recoil) - A state management library for React from Meta.
- [XState](https://github.com/statelyai/xstate) - A library for creating, interpreting, and executing finite state machines and statecharts.
- [Effector](https://github.com/zerobias/effector) - A high-performance, multi-store state manager for handling complex business logic.
- [Immer](https://github.com/immerjs/immer) - A library meant to work with immutable state in a more convenient, mutable way.
- [RxDB](https://github.com/pubkey/rxdb) - A fast, local-first, reactive database for JavaScript applications.

### React Styling
*CSS-in-JS libraries and styling solutions for React components.*

- [TailwindCSS](https://github.com/tailwindlabs/tailwindcss) - A utility-first CSS framework for rapid UI development.
- [Styled Components](https://github.com/styled-components/styled-components) - A popular CSS-in-JS library for styling components using tagged template literals.
- [Emotion](https://github.com/emotion-js/emotion) - A performant and flexible CSS-in-JS library allowing style composition.
- [Vanilla Extract](https://github.com/seek-oss/vanilla-extract) - Zero-runtime stylesheets with full TypeScript type safety.

### React Forms

- [React Hook Form](https://github.com/react-hook-form/react-hook-form) - Performant, flexible, and extensible forms with easy-to-use validation hooks.
- [Formily](https://github.com/alibaba/formily) - A highly performant form solution supporting JSON Schema and dynamic rendering.

- [Tanstack Router](https://github.com/TanStack/router) - A fully type-safe router and state manager for modern React applications.

### Graphics & Animations
*Animation libraries and graphics rendering tools for creating interactive experiences.*

- [Framer Motion](https://github.com/framer/motion) - A production-ready motion library with a simple declarative syntax.
- [React Spring](https://github.com/pmndrs/react-spring) - A spring-physics based animation library for fluid motion.

- [Anime.JS](https://github.com/juliangarnier/anime) - A lightweight JavaScript animation library with a simple API.
- [Auto Animate](https://github.com/formkit/auto-animate) - A zero-config utility that adds smooth transitions to your application automatically.
- [React Tsparticles](https://github.com/matteobruni/tsparticles) - Easily create customizable particle effects like confetti and fireworks.
- [Remotion](https://github.com/remotion-dev/remotion) - A framework for creating videos programmatically using React and CSS.

### React Charts
*Data visualization and charting libraries built for React.*

- [Recharts](https://github.com/recharts/recharts) - A composable charting library built with React components and D3 helpers.
- [Visx](https://github.com/airbnb/visx) - A collection of low-level visualization primitives built by Airbnb.
- [Victory](https://github.com/FormidableLabs/victory) - A set of modular charting components for building interactive data visualizations.
- [Nivo](https://github.com/plouc/nivo) - A rich set of dataviz components built on top of D3 and React.
- [React Flow](https://github.com/xyflow/xyflow) - A highly customizable library for building node-based editors and interactive diagrams.

- [React i18next](https://github.com/i18next/react-i18next) - A powerful internationalization framework for React based on i18next.
- [FormatJS](https://github.com/formatjs/formatjs) - A modular collection of internationalization libraries, including react-intl.

- [React Intlayer](https://github.com/aymericzip/intlayer) - A type-safe, per-component internationalization solution that supports AI translation.

### Tables & Grids

- [React Grid Layout](https://github.com/react-grid-layout/react-grid-layout) - A draggable and resizable grid layout system with responsive breakpoints.
- [React Data Grid](https://github.com/adazzle/react-data-grid) - A feature-rich and performant data grid component similar to Excel.

### React Testing

- [React Testing Library](https://github.com/testing-library/react-testing-library) - Lightweight utilities for testing React components in a way that resembles user interaction.
- [Cypress](https://github.com/cypress-io/cypress) - A complete end-to-end testing framework for anything that runs in a browser.
- [Playwright](https://github.com/microsoft/playwright) - A framework for reliable end-to-end testing across Chromium, Firefox, and WebKit.

### Development Tools
*Build tools, linters, and development utilities for React development workflow.*

- [Vite](https://github.com/vitejs/vite) - A fast build tool and development server that significantly improves the frontend development experience.
- [Storybook](https://github.com/storybookjs/storybook) - A frontend workshop for building, testing, and documenting UI components in isolation.
- [Parcel](https://github.com/parcel-bundler/parcel) - A zero-configuration build tool and bundler for the web.
- [Reactotron](https://github.com/skellock/reactotron) - A desktop app for inspecting React JS and React Native state and network activity.
- [why-did-you-render](https://github.com/welldone-software/why-did-you-render) - A utility that notifies you about potentially avoidable component re-renders.

## ü§ù Contribution

Contributions are welcome! If you know of a React library or tool that should be on this list, please feel free to create a Pull Request.

### How to contribute

1.  **Fork** the repository on GitHub.
2.  **Clone** your fork locally:
    ```bash
    git clone https://github.com/your-username/awesome-react.git
    ```
3.  **Create a new branch** for your feature or addition:
    ```bash
    git checkout -b add-new-library
    ```
4.  **Edit the README.md file**. Please ensure you follow the existing format:
    ```markdown
    - [Library Name](Link to GitHub Repo or Website) - Brief description.
    ```
5.  **Commit** your changes:
    ```bash
    git commit -m "Add [Library Name]"
    ```
6.  **Push** to your branch:
    ```bash
    git push origin add-new-library
    ```
7.  **Submit a Pull Request** through the GitHub website.

### Guidelines

- Please check if the item is already listed before submitting.
- Keep descriptions short and concise.
- Prioritize linking to the GitHub repository unless the official website is significantly more useful.
- Ensure the library is actively maintained and relevant to the React ecosystem.
